# Changelog (Beta)

All notable changes to the beta channel are documented here.
Format follows https://keepachangelog.com/en/1.1.0/

## [1.6.1b] - 2026-07-02

### Added
- Home library view modes: switch between Books, Series, and Authors with the pill switch above the status tabs
- Series view groups your books into series tiles ã tap to open the series, or tap the play button to resume that series where you left off (it continues from the book you last had in progress)
- Authors view groups your books by author ã tap to see all of that author's books
- Change a tile's cover per view: long-press a Series or Author tile to set its own cover from an online search. It changes only that tile ã the member books keep their own covers

### Changed
- The old manual "join books" grouping is fully replaced by the Series system

## [1.6.0b] - 2026-07-02

### Fixed
- Update checker now finds new releases reliably. GitHub lists releases by tag-commit date, not version, so a release cut from an older tag could sort below older ones and be missed entirely ã the app now picks the highest version number instead of trusting the list order, and scans more releases
- The folder picker's Select / Cancel buttons are no longer cut off behind the gesture navigation bar

### Added
- Choose how your library folders are read: Automatic, Author / Series / Book, or Author - Series / Book. Pick it on first launch or in Settings ã Library ã Library structure
- The app now detects when audiobook files or folders are removed from your device: a book whose files are all gone is hidden (kept, with its progress, in case it comes back) and a book that lost only some files is trimmed
- On launch, if an update is available you'll get an update screen with the release notes and Install / Skip. Skipping suppresses only that version, so the next release still prompts you
- Reset library action in Settings ã Library ã clears every book from the app (your audio files on disk are kept) for a clean re-scan

### Changed
- Series are now first-class: the library keeps a proper series record (seeded automatically from your existing series info). This is the groundwork for series covers, a series info screen, series-wide options, and continuous series playback arriving in upcoming beta updates
- Automatic book "joining" has been retired in favour of the new series system

## [1.5.2b] - 2026-06-29

### Fixed
- Pressing Play (or Resume) no longer resets to the start of the current chapter after a book has been completed ã saving a new position now clears the "finished" flag so resume picks up from where you actually are
- Auto-rewind no longer overshoots past the start of a chapter when the saved position within the file is shorter than the rewind amount

## [1.5.1b] - 2026-06-29

### Fixed
- Playback no longer resets to the beginning every time ã position is now correctly restored on resume (fixes a file-sort mismatch and a zero-position overwrite that together corrupted the saved position)
- Opening a book from the home screen no longer triggers multiple simultaneous play() calls when the player screen reloads

## [1.5.0b] - 2026-06-27

### Added
- Per-book listening history: open it from the player's 3-dot menu or the book-info screen to see your listening sessions (with chapter span and time listened) and confirmed chapter/bookmark jumps, as a frosted overlay over the page. Tap a session to jump back to where it ended and resume playing
- Per-book Skip silence: a toggle on the player (in place of the old speed button) that auto-skips silent gaps; tune the minimum silence length and sensitivity in Settings ã Playback
- "This book" reset in the audio settings sheet: clear a book's own speed, boost, or EQ override and fall back to the defaults

### Changed
- Opening a book now morphs its cover: tapping a grid tile, the now-playing card, or the resume card expands the cover into the player's backdrop, and grid taps animate from the tapped book itself
- Chapters now open as a frosted in-player overlay instead of a bottom sheet, with the player blurred behind for legibility
- The What's-new changelog is now rendered as styled version cards with per-category (Added / Changed / Fixed) headers
- The home search button has a clearer, accent-tinted treatment
- New launcher icon: the compass rose now carries an audio "play" mark at its hub

### Fixed
- The seconds label inside the skip-forward / skip-back buttons is now centred in the arrow

## [1.4.0b] - 2026-06-27

### Added
- The home now-playing / resume card now expands upward into the full player with a smooth container-transform animation (and collapses back on return)
- Modern adaptive launcher icon: a compass rose on a deep navy gradient, with a themed-icon (monochrome) layer for Android 13+
- The widget's play button now starts the last-played book from cold ã no need to open the app first; it restores position, speed, and per-book volume boost/EQ

### Changed
- Book info and the player are now a single screen with no navigation between them: tapping Resume crossfades in place, and back / swipe-back from either always returns straight to the home screen (never bounces to the info panel)
- Refresh is now strictly additive: existing books are never re-titled, re-grouped, or reset ã only brand-new folders are added, and new files dropped into an existing book are appended without touching its progress
- Manually joined or split books are now permanently respected; a refresh will never silently re-join a group you split or disturb one you created
- Books are no longer artificially divided into chapters; a file shows chapters only when its own metadata defines them (existing auto-sliced chapters are cleaned up on launch)
- Continued the app-wide reskin onto the player's design language: squircle cards, accent-tinted controls, pill buttons, and list animations across Search, Series, and the Join-books editor

## [1.3.0b] - 2026-06-27

### Added
- Joined group info screen now uses the same full-bleed player backdrop and crossfade layout as the book info screen ã tap Resume to enter the player without a visual jump

- Press-scale feedback on grid cards, nav icons, settings rows, search results, and series rows ã cards slightly shrink on press and spring back with overshoot
- List appearance animation on home grid items (reorder/add/remove animate smoothly)

### Changed

- `BookSettingsSheet` deleted ã player overflow uses the unified `BookOptionsSheet` with a Playback section instead
- Cover backdrop (`ReflectedCoverBackdrop`), scrim controls (`ScrimButton`, `ScrimPill`), and info panel (`BookInfoPanel`) extracted to shared components so every screen using the player design language renders identically
