# Changelog (Beta)

All notable changes to the beta channel are documented here.
Format follows https://keepachangelog.com/en/1.1.0/

## [1.5.1b] - 2026-06-29

### Fixed
- Playback no longer resets to the beginning every time — position is now correctly restored on resume (fixes a file-sort mismatch and a zero-position overwrite that together corrupted the saved position)
- Opening a book from the home screen no longer triggers multiple simultaneous play() calls when the player screen reloads

## [1.5.0b] - 2026-06-27

### Added
- Per-book listening history: open it from the player's 3-dot menu or the book-info screen to see your listening sessions (with chapter span and time listened) and confirmed chapter/bookmark jumps, as a frosted overlay over the page. Tap a session to jump back to where it ended and resume playing
- Per-book Skip silence: a toggle on the player (in place of the old speed button) that auto-skips silent gaps; tune the minimum silence length and sensitivity in Settings → Playback
- "This book" reset in the audio settings sheet: clear a book's own speed, boost, or EQ override and fall back to the defaults

### Changed
- Opening a book now morphs its cover: tapping a grid tile, the now-playing card, or the resume card expands the cover into the player's backdrop, and grid taps animate from the tapped book itself
- Chapters now open as a frosted in-player overlay instead of a bottom sheet, with the player blurred behind for legibility
- The What's-new changelog is now rendered as styled version cards with per-category (Added / Changed / Fixed) headers
- The home search button has a clearer, accent-tinted treatment
- New launcher icon: the compass rose now carries an audio "play" mark at its hub

### Fixed
- The seconds label inside the skip-forward / skip-back buttons is now centred in the arrow

## [1.4.0b] - 2026-06-27

### Added
- The home now-playing / resume card now expands upward into the full player with a smooth container-transform animation (and collapses back on return)
- Modern adaptive launcher icon: a compass rose on a deep navy gradient, with a themed-icon (monochrome) layer for Android 13+
- The widget's play button now starts the last-played book from cold — no need to open the app first; it restores position, speed, and per-book volume boost/EQ

### Changed
- Book info and the player are now a single screen with no navigation between them: tapping Resume crossfades in place, and back / swipe-back from either always returns straight to the home screen (never bounces to the info panel)
- Refresh is now strictly additive: existing books are never re-titled, re-grouped, or reset — only brand-new folders are added, and new files dropped into an existing book are appended without touching its progress
- Manually joined or split books are now permanently respected; a refresh will never silently re-join a group you split or disturb one you created
- Books are no longer artificially divided into chapters; a file shows chapters only when its own metadata defines them (existing auto-sliced chapters are cleaned up on launch)
- Continued the app-wide reskin onto the player's design language: squircle cards, accent-tinted controls, pill buttons, and list animations across Search, Series, and the Join-books editor

## [1.3.0b] - 2026-06-27

### Added
- Joined group info screen now uses the same full-bleed player backdrop and crossfade layout as the book info screen — tap Resume to enter the player without a visual jump
- Parts list shown in the group info panel (title + duration per book)
- Unified book options sheet: the player overflow now opens the same Book Options sheet as the home long-press, with an extra Playback section (speed / boost / gallery cover)
- Spring-physics navigation transitions (OxygenOS/ColorOS 16 feel): screens scale and slide in/out with a natural overshoot on enter
- Press-scale feedback on grid cards, nav icons, settings rows, search results, and series rows — cards slightly shrink on press and spring back with overshoot
- List appearance animation on home grid items (reorder/add/remove animate smoothly)
- `pressScale` modifier and `MotionTokens` spring constants available for reuse across the app

### Changed
- Group info screen is now the player screen in info mode — `GroupInfoScreen` and `GroupInfoViewModel` deleted, logic unified into `PlayerViewModel`
- `BookSettingsSheet` deleted — player overflow uses the unified `BookOptionsSheet` with a Playback section instead
- Cover backdrop (`ReflectedCoverBackdrop`), scrim controls (`ScrimButton`, `ScrimPill`), and info panel (`BookInfoPanel`) extracted to shared components so every screen using the player design language renders identically

## [1.2.9b] - 2026-06-26

### Added
- Cover image search now uses general web image search (DuckDuckGo) instead of audiobook-specific databases — finds any image
- Info screen to player transition: tapping Resume now crossfades only the controls; the cover background stays completely still
- Auto-rewind trigger and rewind amount are now continuous sliders with +/− controls (0–30 min trigger, 0–90 sec rewind)
- Settings → Library: "Refresh all cover effects" button to re-bake the blur background for every book at once

## [1.2.8b] - 2026-06-26

### Added
- Progressive blur cover effect on the player and book info screens: original image with a vertically-flipped reflection below; top 80% of the cover stays sharp, blur ramps up over the final 20% and into the reflection (70% strength at the seam, 100% at the bottom)
- Group info screen: dedicated screen for joined book groups showing cover, progress, and a parts list
- Online cover search now uses iTunes (audiobook-specific) and OpenLibrary instead of Google Images
- Auto-rewind now always triggers after the app is closed and reopened, not just after a long pause

### Changed
- Rewind amount in settings is now a chip row (2 / 5 / 7 / 10 / 15 / 20 s) instead of a slider
- Player back button goes directly to home, skipping the book info screen

## [1.2.6b] - 2026-06-25

### Added
- Redesigned full-bleed player: cover fills the screen with controls over a scrim, chapter context line, and return/confirm jump-history pills
- Library status tabs on the home screen — All / Listening / Not started / Finished, with live counts
- Online cover search (Google Books with OpenLibrary fallback) from a book's options
- Series name + position and reading-status editing in a book's options sheet
- Logical chapters: long files with no embedded chapters are sliced into ~10-minute chapters for a usable table of contents
- Fine-adjust − / + buttons on the speed and volume-boost sliders

### Changed
- Skip buttons now show the actual configured seconds instead of a fixed number
- Tap-and-hold on a book now only selects; the selection bar floats over the top without pushing the grid down, and a 3-dot menu opens book options when one book is selected
- Tapping a home-screen widget opens the active player, not just the app
- The cover-driven theme now carries across the whole app even while idle, using the last-played book's colours
- Folder scanning uses embedded album tags to split loose multi-book folders and to avoid shattering a single tagged book

## [1.2.5b] - 2026-06-22

### Fixed
- Crash on playback start caused by accessing MediaController on the wrong thread in the position ticker

## [1.2.4b] - 2026-06-22

### Fixed
- Seek bar and position time now update continuously during playback — previously only refreshed on pause/resume

## [1.2.3b] - 2026-06-22

### Fixed
- Scan-audiobooks sheet no longer opens on every app launch — only appears on first launch when no folder is set
- Beta builds now check for pre-release updates only; stable builds check for stable releases only (channels no longer cross-notify)

## [1.2.2b] - 2026-06-22

### Added
- Per-book audio settings: EQ and boost are saved per book and restored on playback — settings no longer carry over between books
- Audio presets are now tab-specific: Speed, Boost, and EQ tabs each show and save their own presets
- Long-press a named preset chip to overwrite its stored value with the current slider position (with haptic feedback)
- Home screen shows last-played book as a resume card after app restart, in the same paused-state UI as an active session
- Long hold a book card (1.5 s) opens Book Options: edit title and author, hide from library, or delete permanently
- Hidden books list in Settings → Library — restore ignored books individually
- Changelog shown in Settings → About, per channel
- Update check button moved to Settings → About

### Changed
- Scan folder button removed from nav bar — scanning now happens automatically on first launch and via pull-to-refresh
- First launch automatically prompts for root audio directory
- Header text removed from home screen

### Fixed
- Volume boost no longer bleeds between books (was only saving EQ per-book before)

### Security
- Cover art files are now hidden from the phone gallery via `.nomedia`
